Qanuni — منصة يمنية ذكية للبحث والتحليل القانوني

الرؤية

أن تكون قانــــوني المنصة القانونية الرقمية الأولى في اليمن، تدمج أحدث تقنيات الذكاء الاصطناعي مع الخبرة القانونية لتوفير خدمات البحث والتحليل والصياغة القانونية لكل يمني.

الجمهور المستهدف

  • محامون — للبحث في السوابق القضائية والنصوص القانونية
  • قضاة — للاطلاع على الأحكام والتشريعات
  • باحثون قانونيون — لتحليل النصوص ودراسة القضايا
  • مواطنون — لفهم حقوقهم والوصول إلى المعلومات القانونية

تقنيات المشروع

Backend: Python FastAPI + asyncpg Frontend: Next.js 14 + TypeScript Database: PostgreSQL 16 with Arabic FTS Vector DB: Qdrant + OpenAI Embeddings Cache: Redis 7 OCR: PaddleOCR + Google Vision API AI: GPT-4 Turbo + LangChain Email: SendGrid Reverse Proxy: Caddy + Auto SSL Container: Docker / Docker Compose

٧ الأمان

طبقات أمنية متعددة لحماية المنصة وبيانات المستخدمين

🔑 JWT مع Refresh Tokens

  • Access Token: 30 دقيقة (configurable)
  • Refresh Token: 7 أيام
  • خوارزمية: HS256
  • تحديث تلقائي عبر axios interceptor
  • قائمة انتظار (queue) للطلبات المتزامنة أثناء التحديث
  • تخزين مؤمن في localStorage + httpOnly cookie للمسار

🛡️ Rate Limiting

  • 5 محاولات تسجيل دخول في الدقيقة
  • استخدام Redis كـ back-end للتخزين المؤقت
  • إرجاع 429 Too Many Requests
  • رسالة خطأ عربية: "محاولات كثيرة، حاول بعد دقيقة"

🔒 Security Headers

  • CSP: Content-Security-Policy مقيدة
  • X-Frame-Options: DENY
  • X-Content-Type-Options: nosniff
  • Referrer-Policy: strict-origin-when-cross-origin
  • Permissions-Policy: تقييد الكاميرا والميكروفون
  • Strict-Transport-Security: max-age (فقط HTTPS)

🔄 تدفق تحديث التوكن

  1. الطلب المرسل ← 401 Unauthorized
  2. axios interceptor يلتقط الخطأ ← يتحقق من refresh_token
  3. إن وجد ← إرسال طلب POST /api/v1/auth/refresh

  5. تحديث localStorage + إعادة المحاولة
  6. لو فشل التحديث ← مسح البيانات ← إعادة توجيه لـ /login

١٠ القرارات التقنية (Why?)

شرح أسباب كل قرار تقني مهم في المشروع

🗄️ PostgreSQL Full-Text Search بدلاً من Elasticsearch

القرار: استخدام to_tsvector('arabic') مع فهارس GIN بدلاً من Elasticsearch.

لماذا؟

  • التكلفة: Elasticsearch يتطلب ذاكرة RAM كبيرة (≥4GB). PostgreSQL يعمل داخل 512MB.
  • البساطة: لا حاجة لإدارة كتلة منفصلة. ترحيل واحد (migration) يبني الفهرس.
  • كافٍ للحجم الحالي: خططنا الأولية تغطي آلاف النصوص وليس ملايينها. FTS في PG16 يقدم أداء ممتازاً.
  • استمرارية: بيانات البحث متكاملة مع بقية البيانات في نفس قاعدة البيانات.

⚡ AsyncQdrantClient بدلاً من QdrantClient (sync)

القرار: استخدام AsyncQdrantClient مع await.

لماذا؟

  • FastAPI يعمل على asyncio — استخدام client متزامن (sync) يحجب event loop.
  • يؤدي الحجب إلى تدهور أداء جميع الطلبات الأخرى.
  • AsyncQdrantClient يسمح بمعالجة متزامنة حقيقية لطلبات Qdrant.
  • مهم خاصة أثناء الـ ingestion: رفع مستند + تضمين + إدراج في Qdrant دون حجب باقي API.

📸 PaddleOCR + Google Vision بدلاً من حل واحد

القرار: نظام هجين: PaddleOCR (مجاني محلي) + Google Vision (سحابي مدفوع).

لماذا؟

  • PaddleOCR: سرعة عالية (2-10 ثوانٍ)، يعمل محلياً دون إنترنت، مجاني تماماً.
  • Google Vision: دقة ~95% للنصوص العربية، $1.50/1000 صفحة فقط.
  • التوفير: ~80% من المستندات تكتفي بـ PaddleOCR. الدفع فقط عند الحاجة للدقة العالية.
  • المرونة: المستوى 3 اليدوي يعطي المستخدم تحكماً كاملاً.
  • مستقبلاً: يمكن إضافة Ollama (OCR محلي) كبديل مجاني لـ Google Vision.

🤖 Ollama — بديل OpenAI المستقبلي المجاني

القرار: ترك بنية مرنة تسمح باستبدال OpenAI بـ Ollama محلي.

لماذا؟

  • تقليل تكاليف التشغيل على المدى الطويل.
  • النماذج مفتوحة المصدر (Mistral, Llama, Qwen) تحسنت كثيراً في اللغة العربية.
  • استقلالية تامة عن الخدمات السحابية — مهم لمنصة يمنية.
  • البنية الحالية (LangChain, embeddings, Qdrant) متوافقة مع Ollama.
  • التغيير المطلوب: استبدال ChatOpenAI بـ ChatOllama وتعديل سطر embedding.

📧 SendGrid للبريد الإلكتروني

القرار: استخدام SendGrid API لإرسال الإيميلات.

لماذا؟

  • طبقة مجانية: 100 إيميل/يوم — كافية للمرحلة الحالية.
  • يدعم إرسال HTML templates.
  • API بسيطة عبر httpx — لا حاجة لمكتبات ثقيلة.
  • متاح في اليمن عبر الويب API (ليس SMTP الذي قد يُحظر).

🌐 Caddy بدلاً من Nginx

القرار: استخدام Caddy كـ reverse proxy بدلاً من Nginx التقليدي.

لماذا؟

  • SSL تلقائي: Caddy يحصل على شهادات Let's Encrypt وينشط HTTPS دون أي جهد.
  • Caddyfile: أبسط بكثير من nginx.conf — 10 أسطر بدلاً من 50.
  • أداء: أداء ممتاز مع دعم HTTP/2 و HTTP/3.
  • أمان: إعدادات أمان افتراضية أفضل.

📦 Alembic للترحيلات

القرار: استخدام Alembic لإدارة تغييرات قاعدة البيانات بدلاً من SQL الخام.

لماذا؟

  • تتبع تاريخ التغييرات في قاعدة البيانات.
  • ترقية/تراجع آمن مع upgrade و downgrade.
  • تكامل مع SQLAlchemy async.
  • --autogenerate يكتشف تغييرات النماذج تلقائياً.
  • ضروري لفريق متعدد المطورين — يعرف الجميع حالة قاعدة البيانات.

🔐 JWT + Refresh Tokens بدلاً من Session-based

القرار: JWT (JSON Web Tokens) مع توكن تحديث منفصل.

لماذا؟

  • عديم الحالة (Stateless) — لا حاجة لجلسات خادم.
  • مثالي لـ API REST + تطبيق Next.js (فصل تام).
  • Refresh token يسمح بالبقاء مسجلاً 7 أيام مع أمان جيد.
  • قائمة انتظار (queue) تمنع السباق (race condition) عند تحديث التوكن.
  • middleware في Next.js يقرأ التوكن من الكوكي — حماية المسارات بسرعة.

ما ينتظر قانــــوني في المراحل القادمة

🚀 المرحلة الرابعة: نشر VPS

  • شراء VPS (Digital Ocean / Hetzner / محلي يمني)
  • تثبيت Docker + Docker Compose
  • ربط النطاق (Domain)
  • تفعيل Caddy SSL
  • إعداد المراقبة (monitoring)
  • النسخ الاحتياطي التلقائي (backups)
  • CI/CD pipeline (GitHub Actions)

🧠 دمج Ollama للذكاء الاصطناعي المجاني

  • تشغيل Ollama داخل حاوية Docker
  • نموذج مقترح: Qwen2.5-14B أو Aya-23 (يدعم العربية)
  • جعله اختيارياً — المستخدم يختار OpenAI أو Ollama
  • تقليل التكاليف إلى الصفر للمستخدمين المحليين
  • التعديل: استبدال ChatOpenAI بـ ChatOllama في LangChain

🔑 إعداد Google Vision API Key

  • تفعيل حساب Google Cloud
  • تمكين Vision API
  • إنشاء API Key
  • إضافته إلى .env → GOOGLE_VISION_API_KEY
  • المستوى 2 و 3 سيعملان تلقائياً بعد الإعداد

💳 ربط بوابات الدفع

  • التواصل مع Jeep, Easy, Jawali, Al-Kuraimi
  • الحصول على API keys و merchant IDs
  • تنفيذ webhook لكل بوابة
  • معالجة حالات الدفع (pending, completed, failed, refunded)
  • ترقية الاشتراك تلقائياً بعد الدفع

📚 تغذية المحتوى (Ingestion)

  • جمع النصوص القانونية اليمنية (PDFs, وورد)
  • معالجة OCR للقوانين الورقية

  • تضمينها في Qdrant للبحث الذكي
  • أولويات: الدستور اليمني، القانون التجاري، قانون الأحوال الشخصية، قانون العقوبات
    • إضافة سابقة قضائية من أحكام المحكمة العليا
